# TeamMindAI · 项目总览（必读）

## 这是什么
AI 企业大脑平台（产品名 **TeamMindAI**）：面向初创团队和技术型企业的**多 Agent 智能协作中台**。
它**不是**「AI 版禅道/Jira」，而是一个**信息对齐中台**。区别于传统项目管理工具的核心价值 =
**项目记忆（Memory） + AI 员工（Agents） + 需求对齐（Requirement Alignment）**。

权威来源：`docs/AI企业大脑平台_01_系统SPEC规划文档.md`、`_02_项目架构设计文档.md`、`_03_产品原型文档.md`。
任何业务逻辑、字段、状态流转、页面以这三份文档为准；技术实现以 `docs/AI企业大脑平台_04_技术栈落地说明.md` 为准。

## 核心业务闭环（一切围绕它设计）
```
想法输入 → AI 需求澄清 → 标准需求卡片 → 需求评审 → AI 任务拆解 → 看板推进 → 通知提醒 → 项目记忆沉淀
```

## 五大设计原则（编码时必须遵守）
1. **先对齐，再执行**：需求进入开发前必须补全 背景/目标/范围/不做范围/验收标准。
2. **AI 是员工，不是聊天框**：AI 在需求、任务、会议、报告、通知等场景主动承担职责，而非仅一个聊天窗。
3. **项目空间是核心容器**：所有需求、任务、会议、决策、知识、AI 对话都必须归属到某个 `Project`。
4. **信息要结构化**：AI 输出必须能落库为需求卡片/任务/决策/记忆（用 zod schema + `generateObject`），不能只停留在聊天文本。
5. **关键节点人工确认**：AI 只产出草案；需求确认、任务分配、邮件群发、上线验收等由人类确认。

## MVP 落地优先级（P0 先做）
项目空间 → AI 对齐室 → 标准需求卡片 → 任务看板 → 邮件通知 → 项目记忆。
P1：会议纪要、决策记录、项目报告、知识库、AI 员工管理、通知规则。
P2：企业微信/飞书深度集成、GitLab、项目健康评分、人员负载、自动复盘。

## 第一性判断标准
做任何功能前自问：它是否服务于「把模糊想法变成清晰、可追踪、可沉淀的组织资产」？如果只是又一个任务录入框，请重新设计。
