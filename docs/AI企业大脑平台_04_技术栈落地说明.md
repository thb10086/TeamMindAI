> 本文是**技术实现的权威来源**。业务/字段/状态/页面以 01、02、03 三份文档为准；凡涉及"用什么技术、怎么落地、怎么启动"以本文为准。

---

## 1. 为什么是方案A（统一 TypeScript 全栈）

初版架构文档建议 Vue + Java Spring Boot + Python FastAPI 三语言。考虑到本项目是**面向初创团队的 MVP**，需要：协同开发友好、招人/上手快、AI 能力开箱即用、单仓库单部署，最终选定**统一 TypeScript 全栈、单 Next.js 应用**：

- **一种语言**：前端、后端、AI 编排都是 TS，认知负担最低，最适合小团队 VibeCoding。
- **AI 开箱即用**：Vercel AI SDK 直接支持流式、工具调用、`generateObject` 结构化输出，且原生兼容 OpenAI 兼容网关（AgentLLM）。
- **结构化落库**：zod schema 同时服务于校验、类型推导与 AI 结构化输出，天然贴合"信息要结构化"原则。
- **演进路径清晰**：当 AI 编排复杂到需要独立服务时，可平滑抽出独立服务，业务层不受影响。

> 架构文档 §4.1 本就写明"MVP 不建议一开始过度微服务化"，方案A 与之一致。

## 2. 技术栈总览

| 层 | 选型 | 说明 |
|---|---|---|
| 框架 | Next.js 15 (App Router) + React 19 | 全栈一体，RSC + Server Actions |
| 语言 | TypeScript (strict) | 全程类型安全 |
| 样式 | Tailwind CSS v4 | CSS-first，无需 tailwind.config |
| UI | shadcn/ui (new-york, neutral) + lucide-react | 可复制、可定制的组件 |
| ORM/DB | Prisma + PostgreSQL + pgvector | 结构化数据 + 向量检索同库 |
| AI | Vercel AI SDK v6（`ai` / `@ai-sdk/openai-compatible` / `@ai-sdk/react`）| 统一走 AgentLLM |
| 认证 | Auth.js (next-auth v5) | 凭证登录 + 可扩展 OAuth |
| 异步/定时 | BullMQ + ioredis | 通知、延期扫描、周报 |
| 对象存储 | MinIO（S3 兼容） | 附件/文档 |
| 校验 | zod | 输入校验 + AI 输出 schema |

## 4. AgentLLM 接入（OpenAI 兼容网关）

- base_url：`https://agentllm.linkyun.co/v1`；鉴权 `Authorization: Bearer <AGENTLLM_API_KEY>`。
- 统一封装在 `src/lib/ai/provider.ts`，业务代码用 `agentllm(modelId)` 取对话模型，不要散落 fetch/base_url。
- 模型档位（可在 `.env` 调整，完整列表 `GET /v1/models`）：
  - `chat`（默认 `claude-sonnet`）：强指令跟随，用于需求澄清、任务拆解等结构化场景。
  - `fast`（默认 `gemini-3.5-flash`）：快/省，用于摘要、分类、提醒文案。
  - `embedding`（`text-embedding-3-large`）：固定降维 **1536**（pgvector 索引上限 2000）。
- 网关亦支持图像/TTS/ASR（会议纪要、生图等可后续接入）。

**对话（流式）**：`streamText({ model: agentllm(MODELS.chat), system, messages }).toUIMessageStreamResponse()`，前端用 `@ai-sdk/react` 的 `useChat` + `DefaultChatTransport`。

**结构化落库**：`generateObject({ model, schema: RequirementDraftSchema, ... })`，输出直接落 `requirement` 表。

## 5. 数据模型与向量

- 模型定义见 `prisma/schema.prisma`，覆盖 02 文档 §8 全部核心对象 + Auth.js + AI 对话持久化。
- 主键用 `cuid()`；状态用枚举（`RequirementStatus`/`TaskStatus`/…）。
- 向量列：`memory.embedding`、`knowledge_chunk.embedding` 为 `vector(1536)`，需用原生 SQL（`$queryRaw`）做相似检索。
- 检索策略（02 §7.3）：结构化条件 + 全文 + 向量相似 + 时间/重要性权重。

## 6. 本地启动

```bash
cp .env.example .env          # 填入 AGENTLLM_API_KEY（其余可用默认）
npm install
npm run infra:up              # 启动 postgres(pgvector)/redis/minio
npm run db:push               # 建表

```

## 7. 借鉴的开源最佳实践

- **shadcn/ui**：组件不是 npm 黑盒，而是复制进仓库可改，契合定制需求。
- **Vercel AI Chatbot / AI SDK 范式**：流式对话、`useChat`、结构化输出的事实标准。
- **Plane（开源 Jira 替代）**：看板/项目/任务数据建模与状态流转参考。
- **dnd-kit**：现代、无障碍的看板拖拽。
- **Auth.js**：认证最佳实践，RBAC 之上叠加项目级权限。
- **BullMQ**：通知、延期扫描、定时周报的可靠队列。

> 原则：能用成熟开源就不自造轮子；自研只发生在"AI 员工编排 + 项目记忆"这类核心差异化能力上。
